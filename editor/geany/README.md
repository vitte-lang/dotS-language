# Geany Runtime Setup (DotS)

Source of truth: all Geany assets live under `editor/geany/*`.
Legacy copies under `filedefs/*`, `tools/geany/*`, and `dotS-language*.geany` are generated by sync script.

## Quickstart (30s)

```bash
bash scripts/geany_runtime_bootstrap.sh
```

## Exact commands

Install/update:

```bash
bash scripts/install_geany_runtime.sh
bash scripts/geany_sync_runtime_profile.sh
```

Verify:

```bash
bash scripts/geany_runtime_doctor.sh
bash scripts/ci_geany_assets_gate.sh
```

Uninstall/rollback:

```bash
bash scripts/uninstall_geany_runtime.sh
```

## Runtime profiles

- `editor/geany/dotS-language.runtime.geany`
- `editor/geany/dotS-language.strict-runtime.geany`

Strict env profile uses:
- `DOTS_RUNTIME_STRICT=1`
- `DOTS_CLI_STRICT=1`
- `DOTS_RUNTIME_SEED=424242`

## Security policy

- Geany tools must call local scripts only.
- No direct network commands (`curl`, `wget`, raw URLs).
- No shell-risky runtime patterns in assets.

## Performance notes

For large `runtime/*` trees:
- keep symbol regex narrow (`proc|form|entry|const`)
- prefer strict runtime profile for deterministic reproductions
- use `Run only changed tests/runtime` task first

## Troubleshooting

- LSP plugin missing: disable GeanyLSP fallback still works (`geany_goto_definition.sh`, `geany_find_references.sh`).
- `dots` not found: export PATH so `dots` is visible to Geany terminal.
- Click-to-line not working: validate compiler regex in `filetypes.DOTS.conf`.
- Snippets not visible: verify `~/.config/geany/snippets.conf` contains DOTS/VIT sections.
- Templates not visible: verify `~/.config/geany/filedefs/templates/files/*` present.

## FAQ: Geany runtime vs Vim/Emacs

- Format current file: Geany tool (`dots fmt --write %f`) vs `:DotsFmt` / `before-save-hook`.
- Goto-definition: Geany LSP or `rg` fallback vs LSP in Vim/Emacs.
- Runtime doctor/status: same CLI commands across editors.
