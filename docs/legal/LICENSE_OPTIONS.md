# License Options (Suggestions)

If licensing strategy evolves, common options are:

1. GPL-3.0-or-later (current)
- Strong copyleft, reciprocal distribution obligations.

2. AGPL-3.0-or-later
- Extends copyleft obligations to network/SaaS distribution contexts.

3. LGPL-3.0-or-later
- Weaker copyleft, often used for libraries.

4. MPL-2.0
- File-level copyleft, more permissive for mixed codebases.

5. Apache-2.0
- Permissive with explicit patent grant.

6. MIT
- Very permissive, minimal obligations.

Dual-license patterns can be considered (e.g., GPL + commercial).

Important:
- License changes require contributor/copyright analysis.
- Validate strategy with legal counsel before switching.
